# CLAUDE.md — Claude Code 프로젝트 지침서

## 프로젝트 개요

CoinGecko API를 활용한 암호화폐 트렌드 조회 및 대시보드 웹 애플리케이션이다. Next.js 15 (App Router) 기반으로, 트렌드 코인 목록 조회, 즐겨찾기, 코인 상세 정보(차트·커뮤니티 반응) 대시보드 기능을 제공한다.

## 주의사항

### 민감 파일 — 절대 읽거나 수정하지 않는다
- `.env`, `.env.*` (환경변수 파일)
- `*.pem` (인증서/키 파일)
- `src/service/coingecko/coingeckoService.ts` 내 API 키 값을 외부에 노출하지 않는다

### Lock 파일 — 직접 수정하지 않는다
- `package-lock.json`
- `pnpm-lock.yaml`
- `yarn.lock`

### 빌드 산출물 — 수정하지 않는다
- `node_modules/`
- `.next/`
- `build/`
- `out/`

## Git 워크플로우

- 브랜치 전략: Git Flow (main / develop / feature / hotfix / release)
- 작업 완료 후 반드시 commit + push까지 진행한다
- 커밋은 논리적 단위로 나눈다 (설정 변경 / 기능 구현 / 버그 수정 / 리팩토링 등 분리)
- 커밋 메시지 규칙:
  - prefix를 반드시 사용한다: feat / fix / chore / refactor / style / docs / test
  - 본문은 한국어로 작성한다
  - 형식: {prefix}: {한국어 설명}
  - 예시: feat: 로그인 페이지 UI 구현
- push 전에 현재 브랜치를 확인하고, main 또는 develop에 직접 push하지 않는다
- PR은 최소 1명의 승인이 필요하다

## 코드 수정 원칙

- 코드를 수정하기 전에 반드시 기존 코드를 먼저 읽고 패턴을 파악한다
- 한 번에 너무 많은 파일을 수정하지 않는다. 변경 범위를 최소화한다
- 요청받은 범위 밖의 코드를 임의로 리팩토링하지 않는다
- 확실하지 않은 부분은 추측하지 말고 질문한다
- 코드 수정 후 프로젝트에 설정된 린트/포맷터/타입체크가 있다면 반드시 실행하여 오류가 없는지 확인한다
- 작업 완료 시 변경 사항을 요약하여 알려준다

## Bash 명령어 실행 원칙

- 여러 명령어를 실행할 때 `&&`나 `;`로 체이닝하지 않고, 각 명령어를 개별 Bash 호출로 분리하여 실행한다
- 독립적인 명령어는 병렬로 호출하고, 의존성이 있는 명령어는 순차적으로 호출한다
